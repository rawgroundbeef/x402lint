## v1 Requirements

### Input

- [x] **INP-01**: User can enter a URL and submit for validation
- [x] **INP-02**: User can paste raw JSON into a textarea and submit for validation
- [x] **INP-03**: User can toggle between URL and JSON input modes via tabs — *adjusted: auto-detection instead of tabs*
- [x] **INP-04**: User can load an example valid x402 config with one click
- [x] **INP-05**: Tool fetches URL via Cloudflare Worker proxy to bypass CORS

### Validation

- [x] **VAL-01**: Tool validates x402Version field exists and equals 1
- [x] **VAL-02**: Tool validates payments array exists and has at least one entry
- [x] **VAL-03**: Tool validates each payment has required fields (chain, address, asset, minAmount)
- [x] **VAL-04**: Tool validates chain is one of: base, base-sepolia, solana, solana-devnet
- [x] **VAL-05**: Tool validates EVM addresses using checksum (EIP-55 format, 42 chars starting with 0x)
- [x] **VAL-06**: Tool validates Solana addresses using Base58 format (32-44 chars)
- [x] **VAL-07**: Tool validates asset is valid for chain (USDC/ETH/USDT for EVM, USDC/SOL for Solana)
- [x] **VAL-08**: Tool validates minAmount is a positive decimal number
- [x] **VAL-09**: Tool validates optional fields when present (facilitator.url is HTTPS, maxAmount >= minAmount)
- [x] **VAL-10**: Tool distinguishes errors (blocking) from warnings (recommendations)

### Results

- [x] **RES-01**: Tool displays clear pass/fail status badge after validation
- [x] **RES-02**: Tool displays specific error messages identifying the field and how to fix it
- [x] **RES-03**: Tool displays success confirmation with config summary when valid
- [x] **RES-04**: User can copy validated/formatted config to clipboard with one click
- [x] **RES-05**: Tool formats and beautifies JSON output for readability

### UX

- [x] **UX-01**: Tool shows loading state while fetching URL
- [x] **UX-02**: Tool is responsive and usable on mobile devices
